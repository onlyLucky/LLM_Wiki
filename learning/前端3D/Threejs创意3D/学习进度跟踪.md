#### 费曼学习法

**概念：动画循环 + delta time**

**AI 用大白话讲**：
> 想象你在翻书动画——在便利贴角落画一个小人，每页画一个稍微不同的姿势，快速翻起来就像在动。动画循环就是你的手指每秒翻多少页。delta time 就是你翻这一页花了多少时间。每次翻页，小人移动的距离 = 速度 × 这一页花了多少时间。这样不管翻快翻慢，小人 1 秒内移动的距离是一样的。

**你的理解**：三个转动速率不一致的钟表，delta time = 指针在每次之间移动的距离，不管转动频率多快，每秒移动距离都一样。

**AI 诊断**：
- ✅ 讲对了：三个设备帧率不同、最终效果要一致
- ❌ 小错误：delta time 是时间间隔，不是距离
- ⚠️ 讲漏了：距离 = 速度 × delta time 这个公式

#### 苏格拉底问答

**Q1：为什么切换到单个灯光时需要添加辅助环境光？**
- **你的回答**：辅助环境光确保场景不会全黑，提供全局均匀照明让暗部也能看到物体轮廓。没有的话，点光源只能照亮前方区域，背面和远处纯黑。
- **评分**：10/10
- **哪里对**：理解准确，解释清晰
- **哪里不完整**：无

**Q2：阴影三要素及排查顺序**
- **你的回答**：三要素：灯光.castShadow（有没有能力投射）、物体.castShadow（会不会挡住光）、地面.receiveShadow（能不能显示阴影）。排查顺序：渲染器→灯光→物体→阴影范围→分辨率→物体位置。
- **评分**：10/10
- **哪里对**：三要素理解准确，排查顺序逻辑清晰
- **哪里不完整**：无

**Q3：shadow.mapSize 的效果与限制**
- **你的回答**：阴影贴图分辨率，越大越清晰但越耗性能。常见值：512（低）、1024（中）、2048（高）、4096（极高）。
- **评分**：10/10
- **哪里对**：理解分辨率与质量的关系
- **哪里不完整**：初次回答没有解释"为什么"耗性能
- **补充回答**：shadow.mapSize 翻倍，长宽各翻倍，像素总数增加 4 倍

**Q4：为什么加载 Suzanne 后要遍历并手动添加材质？**
- **你的回答**：Suzanne 是 Blender 内置测试模型，导出 GLB 时只有几何体没有材质信息。GLTF 模型可能包含多个 Mesh，遍历才能逐个设置材质，否则显示为白色或黑色。
- **评分**：10/10
- **哪里对**：理解准确
- **哪里不完整**：无

**Q5：Helper 可见但灯光效果不对，说明什么？**
- **你的回答**：Helper 可见说明灯光位置/方向/范围可视化正确；效果不对通常是参数问题（强度、颜色、材质）或阴影配置问题（三要素、范围、分辨率）。
- **评分**：9/10
- **哪里对**：理解 Helper 的作用，排查思路正确
- **哪里不完整**：Helper 可见只说明 Helper 对象被添加并更新，不代表灯光参数正确

**Q6：物体超出 shadow.camera 范围会怎样？**
- **你的回答**：物体超出范围就没有阴影。解决方法：扩大范围、移动物体、移动灯光。但范围太大会导致阴影模糊，需要在覆盖范围和质量之间平衡。
- **评分**：10/10
- **哪里对**：理解准确，提到了范围与质量的权衡
- **哪里不完整**：无

**Q7：PCFSoftShadowMap 与 BasicShadowMap 的区别**
- **你的回答**：BasicShadowMap 最快但硬边锯齿；PCFShadowMap 中等质量；PCFSoftShadowMap 高质量柔和软边。
- **评分**：10/10
- **哪里对**：三种类型描述准确
- **补充回答**：PCFSoftShadowMap 通过采样周围多个像素并计算"在阴影中的百分比"，让阴影边缘渐变，实现柔和软边

## 五、速查表

### Three.js 速查表（第 1 课更新）

**一句话定义**：Three.js 是 WebGL 的高级封装库，用 JS 创建和控制 3D 场景。

**核心概念**：
| 概念 | 一句话解释 | 类比 |
|------|-----------|------|
| Scene | 3D 场景的容器，放物体、灯光、背景 | 舞台 |
| Camera | 决定观众看到什么（位置、角度、范围） | 摄像机 |
| Renderer | 把 3D 场景计算成 2D 像素画到 canvas | 投影仪 |
| Delta Time | 两帧之间的时间间隔（秒） | 节拍器 |
| Pixel Ratio | 1 个 CSS 像素 = 几个物理像素 | 分辨率倍数 |

**常见错误**：
| 错误 | 为什么会犯 | 正确做法 |
|------|-----------|---------|
| 画面全黑 | 相机在物体内部（z=0）| camera.position.z = 5 或更远 |
| Retina 屏模糊 | 没设 pixelRatio | renderer.setPixelRatio(Math.min(devicePixelRatio, 2)) |
| 窗口缩放变形 | 只改 setSize 没改 aspect | 先改 aspect + updateProjectionMatrix，再 setSize |
| 动画速度不一致 | 直接 += 0.01 没用 delta | 乘以 delta（秒）|
| GPU 内存泄漏 | 没调 dispose() | 用 registerDisposable 注册，统一清理 |

**使用前检查清单**：
- [ ] Scene、Camera、Renderer 都创建了吗？
- [ ] Camera 位置设对了吗？（不在物体内部）
- [ ] setPixelRatio 设了吗？（Math.min(..., 2)）
- [ ] 窗口自适应顺序对吗？（aspect → updateProjectionMatrix → setSize）
- [ ] dispose() 注册了吗？

**自测问题**：
1. Scene、Camera、Renderer 三者各自的职责是什么？
2. requestAnimationFrame 的回调参数是什么？为什么要除以 1000？
3. window 自适应时，为什么必须调 updateProjectionMatrix()？
4. setPixelRatio 设成 10 会有什么后果？
5. 如果不调 dispose()，Three.js 的哪些资源会泄漏？
