# AI 加速学习的六种方法

> 核心思路：把 AI 当成一个随叫随到的私教、考官、速记员和资源筛选器，而不是一个搜索引擎。

---

## 1、分层级：五级成长地图

学任何技能之前，先让 AI 画出从零到独立做项目的五级路径。每一级明确三件事：**该掌握什么、常见错误是什么、达到什么标准**。

### 用法

向 AI 提问：

```
我要学习 [技能名]，请帮我把它分为五个等级：
L1 零基础入门 → L5 能独立交付项目。
每一级列出：
1. 核心知识点（不超过 5 个）
2. 新手最常见的 3 个错误
3. 判断"过关"的硬指标（可量化或可验证）
```

### 为什么有效

- 知道自己在哪一级，才能找到下一步该走的路
- 常见错误提前列出来，避免重复踩坑
- 过关标准让你不再"感觉自己会了"，而是"证明自己会了"

## 2、二八法则：20 小时速成计划

20 小时不多，但用对地方足够入门一个新领域。让 AI 帮你找出**最重要的 20% 内容**，设计成 10 次学习计划。

### 用法

```
我有 20 小时要入门 [技能名]。
请找出最核心的 20% 内容（覆盖 80% 实战场景），
设计 10 次学习计划，每次 2 小时，每次必须包含：
1. 主题和目标
2. 学习资源（1-2 个精选）
3. 动手练习（必须写代码/做题/实操）
4. 复盘清单（3 个自测问题）
```

### 时间分配建议

| 阶段 | 次数 | 内容 |
|------|------|------|
| 基础概念 | 第 1-3 次 | 核心术语、基本操作、最小可运行 demo |
| 核心技能 | 第 4-7 次 | 覆盖 80% 场景的关键能力 |
| 综合实战 | 第 8-10 次 | 做一个小项目，查漏补缺 |

## 3、刨根问底：苏格拉底式问答

一次只问一个问题，从最简单的开始，逐步加深。AI 扮演考官角色。

### 用法

```
你是一个严格的考官。我正在学习 [具体知识点]。
规则：
1. 一次只问我一个问题
2. 从最基础的概念开始，根据我的回答逐步加深
3. 我回答后，你必须：
   - 打分（1-10）
   - 指出哪里答对了
   - 指出哪里答错了
   - 指出哪里不完整
   - 用更准确的方式重新解释我没掌握的部分
4. 然后再出下一题
```

### 进阶用法

- **错题重练**：把之前答错的问题攒起来，隔一天再测
- **角色互换**：让 AI 问你问题后，你反过来给 AI 讲解，AI 帮你找漏洞
- **压力测试**：要求 AI 加入追问"为什么？""还有呢？""如果 X 情况呢？"

## 4、速查表：一页纸压缩术

每学完一个阶段，让 AI 把知识压缩成一张速查表。面试前、考试前、开会前拿出来过一遍，10 分钟理清结构。

### 用法

```
我把 [某阶段] 的学习内容学完了。
请帮我压缩成一张速查表，格式如下：

## [主题名] 速查表

### 一句话定义
用一句话说清楚这是什么。

### 核心概念（不超过 5 个）
每个概念一行：术语 → 一句话解释 → 一个类比

### 真实例子（3 个）
来自实际工作/开源项目的例子，不要教科书式的。

### 常见错误（5 个）
错误描述 → 为什么会犯 → 正确做法

### 使用前检查清单
- [ ] 检查项 1
- [ ] 检查项 2
- ...

### 自测问题（5 个）
不给答案，自己想。
```

### 什么时候用

- 学完一个章节后立刻做
- 隔一周后回顾，看哪些还能直接答出来
- 实战前快速过一遍

## 5、资源筛选：7 天学习路径

资源太多反而不知道看什么。让 AI 从海量信息中筛出**最值得看的 5 个资源**，配一条 7 天路径。

### 用法

```
我要在 7 天内入门 [技能名]。
请帮我：
1. 筛选最值得看的 5 个资源（不要超过 5 个，贪多嚼不烂）
2. 每个资源标注：
   - 适合谁（零基础 / 有编程基础 / 有相关经验）
   - 难度（入门 / 进阶 / 高级）
   - 怎么用（通读 / 跳读 / 只看某几节 / 边看边做）
3. 排成 7 天学习路径，每天明确：
   - 看什么
   - 做什么练习
   - 当天结束时应该能做什么
```

### 资源质量标准

AI 筛选资源时，你可以在 prompt 里加上：

```
优先推荐：
- 有动手环节的（不只是看）
- 社区活跃、有人维护的
- 有真实项目案例的
排除：
- 纯理论无实践的
- 过时超过 2 年的（除非是经典）
- 需要付费且没有试看的
```

## 6、费曼学习法：用大白话讲明白

费曼学习法的核心就一句话：**如果你不能用简单的话解释清楚，说明你还没真正理解。**

### 用法

```
请用一个 10 岁孩子能听懂的方式，给我解释 [概念名]。
用生活中的类比，不要用任何专业术语。
```

等 AI 讲完后，**用自己的话讲回去**，然后让 AI 诊断：

```
这是我用自己的话解释 [概念名] 的内容：
[你的解释]

请帮我分析：
1. ✅ 哪里讲对了
2. ❌ 哪里讲错了
3. ⚠️ 哪里讲漏了
4. 🔄 哪里混淆了两个不同的概念
5. 📌 哪些词看起来高级但我其实没真懂

然后，针对我薄弱的部分，再用更简单的方式解释一次。
```

### 循环流程

```
AI 用大白话讲 → 你用自己的话讲回去 → AI 诊断 → 针对薄弱点再讲 → 你再讲 → ...
```

一般 **2-3 轮**就能把一个概念真正吃透。

---

## 六种方法的组合用法

不必每次学习都用全部六种。根据场景选择：

| 场景 | 推荐方法 |
|------|----------|
| 刚接触一个新领域 | 1 分层级 + 2 二八法则 + 5 资源筛选 |
| 准备面试/考试 | 3 刨根问底 + 4 速查表 |
| 想真正理解一个概念 | 6 费曼学习法 + 3 刨根问底 |
| 时间紧迫要速成 | 2 二八法则 + 5 资源筛选 |
| 长期学习某个技能 | 全部六种，按阶段穿插使用 |
